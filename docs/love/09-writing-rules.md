# Writing Rules（表示規約）

Text Bank（Bundle文・StyleのHigh/Mid/Low文・TendencyのHigh/Mid/Low文）およびResultDocumentを最終的な文章へ整える際の表示規約。ドメイン（値を決める）とプレゼンテーション（どう見せるか）の分離（[00-overview.md](00-overview.md)）における、プレゼンテーション側の文書である。

Composerはこの規約を適用しない（Composerは文言バンクからIDで選ぶだけ。[06-composer.md](06-composer.md)）。この規約が適用されるのは、①Text Bankの文言を執筆する時点、②ResultDocumentを最終出力へ整形する時点、の2箇所である。

## 規約

1. **高・中・低のいずれも肯定的に表現する**。Lowを欠点として書かない（例：行動主導性Low＝「受け身で消極的」ではなく「じっくり相手を見てから動く」。自立性Low＝「依存的」ではなく「相手と過ごす時間を何より大切にする」）
2. **能力ではなく傾向として記述する**
3. **性格を断定しない**
4. **優劣を示す表現を避ける**。High/Mid/Lowは母集団上の位置であり、良し悪しではない（[08-normalizer.md](08-normalizer.md)）。スコアの高低・総得点の比較を「強い/弱い」として表現しない
5. **相手（診断結果の読み手）を否定する語を使わない**
6. **Primitiveの名称は常に一行説明とセットで表示する**（例：「情動性──感情を深く受け取り、自分の想いも強く動く力」）。特に「情動性」は日常語彙として馴染みが薄く、単語だけでは意味が伝わらないため
7. **Tendencyは「〜の傾向があります」を基本形とする**。Tendency（結婚志向・浮気耐性）は性格だけでは決定できない概念であり（[05-tendency.md](05-tendency.md)）、価値観・経験・年齢の影響を受けるため、Styleより一段強く断定を避ける

## 最終整形の責務

接続詞・文体・冗長さ・改行の調整は、Composerの出力（ResultDocument）を受けた後の整形工程が担う。この整形は文言の意味を変えない範囲に限る（意味を変えたい場合はText Bankの文言自体を修正する）。
