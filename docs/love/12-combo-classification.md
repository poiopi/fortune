# 組み合わせ分類仕様（MBTI×血液型 64パターン）

> Love Content Platform v1.2完了後、次の展開として着手する「組み合わせページ自動生成プロジェクト」の基盤仕様。着手前提として[00-overview.md](00-overview.md)〜[11-articles.md](11-articles.md)を前提知識とする。

## 1. 背景・目的

MBTI×血液型の組み合わせコンテンツは、「MBTI×血液型 64パターン完全攻略」等の競合が既に存在する（2026-07-12、Web検索で確認）。単純な「ENTPはこう、A型はこう」という一般論の掛け合わせでは、後発かつ量で劣る当サイトが検索で勝つのは難しい。

Love Engineが他サイトにない資産として持つのは、**9216件のGolden Master実測データ**であり、その中でも特に「MBTI由来のPrimitive傾向と血液型由来のPrimitive傾向が競合したとき、どちらが優勢になるか」という現象は、実測データを持つLove Engineでしか説明できない。この現象を64パターン全体で機械的に分類し、各組み合わせページの核となる説明軸にする。

## 2. 4分類の定義

各MBTIタイプ・各血液型には、それぞれ単体で見たときの「主軸Primitive」（576件・2304件それぞれの中で最も出現率が高いPrimitive）がある。組み合わせたとき（144件）の実際の主軸Primitiveが、両者のうちどちらと一致するか、あるいはどちらとも一致しないかで、以下の4つに分類する。

| 分類 | 条件 |
|---|---|
| **協調型** | 組み合わせの主軸が、MBTI単体の主軸・血液型単体の主軸の**両方**と一致 |
| **拮抗型（血液型優勢）** | 組み合わせの主軸が、血液型単体の主軸と一致し、MBTI単体の主軸とは不一致 |
| **拮抗型（MBTI優勢）** | 組み合わせの主軸が、MBTI単体の主軸と一致し、血液型単体の主軸とは不一致 |
| **転換型** | 組み合わせの主軸が、MBTI単体の主軸・血液型単体の主軸の**どちらとも不一致**（第3のPrimitiveが台頭） |

**分類は数値の閾値を持ち込まず、エンジンの出力（主軸Primitiveの一致・不一致）のみで機械的に決まる。** 出現率の高低（例：40%未満なら「弱い」等）は分類基準に使わない。

### 2-1. アルゴリズム

```
mP = MBTIタイプ単体（576件）での最多主軸Primitive
bP = 血液型単体（2304件）での最多主軸Primitive
cP = 組み合わせ（144件）での最多主軸Primitive

if cP == mP かつ cP == bP:
    協調型
elif cP == bP （かつ cP != mP）:
    拮抗型（血液型優勢）
elif cP == mP （かつ cP != bP）:
    拮抗型（MBTI優勢）
else:
    転換型
```

**注意（実装時の落とし穴）**：初期実装では `mP == bP` を協調型の条件にしていたが、これは誤り。`mP == bP` であっても、組み合わせた実際の結果（cP）が両者と異なる第3のPrimitiveになるケースが4件存在した（後述）。協調型は必ず「組み合わせの実際の結果」で判定すること。

## 5. 転換型（例外ケース）4件の解説

いずれもMBTI単体・血液型単体のどちらの主軸とも異なる、第3のPrimitiveが組み合わせの主軸になるケース。

- **ESTJ×O型・ESTJ×AB型**：ESTJはE（行動力→ACT）・T（挑戦→ACT）・S（安定性→REL）・J（責任感→REL）を持ち、MBTI単体ではACT・REL がスコア上拮抗する構造（単体集計ではRELがわずかに優勢＝53.1%）。O型・AB型はどちらも血液型主軸がSEN（情動性）であり、REL側への加勢が働かない。その結果、星座からの寄与も含めた総合計算で、ACTが組み合わせの主軸として浮上する。
- **INTP×B型・ESFP×B型**：MBTI単体・血液型単体がどちらもREL主軸だが（一見「協調型」に見える）、実際に組み合わせるとTRA（変化志向）が主軸になる。これはB型がTRAIT_CHANGE（変化志向）への寄与を持つ（`inc/blood-trait-mapping.php`）ため、単体では隠れていたTRA寄与が、特定のMBTIタイプと組み合わさったときに顕在化する例。**「両方の入力が同じ方向を指しているように見えても、実際の結果は別の軸に転ぶことがある」**という、見かけの一致だけでは判断できない現象を示す実例。

## 5-1. 主軸集中度の5段階評価

各組み合わせの「組合せ主軸%」（3節の表の列）は、64通り全体で見ると39.6%〜100%まで分布する。この値をそのまま「◯位/64」という順位で見せても読者には直感的でないため、64通りの実測値を5等分した5段階のラベルに変換して提示する。閾値は64通りの実測分布から機械的に算出したもので、恣意的な基準は使わない。

| ラベル | 集中度の範囲（64通り実測の5等分） |
|---|---|
| 集中しやすい | 97.9%超（上位20%） |
| やや集中しやすい | 88.9%超〜97.9%以下（上位20〜40%） |
| 平均的 | 68.1%超〜88.9%以下（上位40〜60%） |
| 分散しやすい | 58.3%超〜68.1%以下（上位60〜80%） |
| 非常に分散しやすい | 58.3%以下（下位20%） |

各記事では、この5段階ラベルを主表示にし、「64通り中◯位」という順位は補足情報として添える（順位だけでは高低が直感的に伝わらないため）。

**「9216件中◯件（◯%）」という表現は使わない**——MBTI×血液型64通りは、どの組み合わせも9216件中144件（1.56%）で完全に同一の値になるため、情報価値がない（ENTP×A型のリファレンス記事執筆時に気づき、削除した）。

## 6. 執筆時の表現方針

「勝った」「負けた」という対戦・勝敗の言葉は使わない。Love Engineは入力を比較して優劣をつけているのではなく、Trait→Primitiveへのスコア加算という機械的な計算をしているだけであるため、**エンジンの処理として説明する**。

- ❌「A型がENTPに勝ちました」
- ✅「A型由来の誠実性が組み合わせの主軸として採用され、ENTP由来の行動主導性は副軸として残りました」

これは[09-writing-rules.md](09-writing-rules.md)の「性格を断定しない」原則の延長でもある。

- 分類：**拮抗型（血液型優勢）**
- MBTI単体：ACT 52.3% ／ 血液型単体：REL 95.5%（血液型単独では最も濃い偏り）／ 組み合わせ：REL 75.0%
- 集中度5段階評価：**平均的**（64通り中29位、5-1節の基準による）
- 副次的な発見：誠実性が主軸になったケース（108件）のうち85.2%（92件）で、行動主導性が副軸として残る。「主軸を譲っても入力の影響が完全には消えない」ことを示す数値で、記事の結論部で最も強調している

**横断レビュー（1本目）で確認・反映した改善点**（2026-07-12、Managerレビュー）：
- 冒頭の結論文は「この組み合わせは『◯◯型』です」という分類の明言から始める（読者が一目で分類を把握できるようにする）
- 「9216件中◯件」という、64通り共通で意味を持たない数値は使わない（5-1節参照）
- 副軸として残る痕跡（このケースでは85.2%）は、単なる補足ではなく記事の主要な差別化ポイントとして前面に出す

## 7-1. テンプレート監査：転換型パイロット（ESTJ×O型）実装結果

63記事への横展開前に、テンプレートが最も構造の異なる「転換型」（4件しかない例外分類）でも成立するかを検証するため、`articles/love/mbti-blood/estj-o/index.php`を試作した（2026-07-12）。

**検証結果：テンプレートは無変更のまま成立した。** `causalRows`（因果テーブルの行データ）は元々可変長の配列だったため、拮抗型・協調型で使っていた2行（MBTI単体・血液型単体）に加えて3行目「組み合わせの結果」を追加するだけで、転換型特有の「どちらの単体主軸でもない第3のPrimitiveが採用される」という構造を、既存の`.win`（採用行のハイライト）スタイルがそのまま正しく機能する形で表現できた。テンプレートのコード変更は不要だった。

- ESTJ×O型の分類：**転換型**。MBTI単体：誠実性53.1%／血液型単体：情動性40.0%／組み合わせ：行動主導性81.3%
- 因果の実体：ESTJは内部でE・T（行動主導性）とS・J（誠実性）が僅差で拮抗する構造を持つ。O型も「行動的」（行動主導性+2）と「おおらか」（情動性+2）が同じ強さで拮抗する構造を持つ。この2つの「僅差の拮抗」が組み合わさると、両者に共通する行動主導性への寄与が積み重なり、単体では優勢でなかった行動主導性が81.3%まで押し上げられる
- 集中度5段階評価：**平均的**（64通り中34位）——「転換型という分類の珍しさ（4/64）」と「集中度の高さ（81.3%は平均的）」は別の軸であることを記事内で明記した
- prev/next：entp-a→estj-oの順で連結

**結論**：転換型4件（ESTJ×O型・ESTJ×AB型・INTP×B型・ESFP×B型）は、causalRowsに3行目を追加する形式で問題なく執筆できる。63記事の自動生成時も、分類が「転換型」の場合のみ3行版、それ以外は2行版という条件分岐をデータ生成スクリプト側で持たせればよい（テンプレート側の分岐は不要）。
