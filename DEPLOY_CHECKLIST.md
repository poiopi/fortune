# デプロイチェックリスト

## デプロイ前

- [ ] 反映対象ファイル一覧（新規・更新・削除）を確認
- [ ] 新規ディレクトリが本番に存在するか確認
- [ ] 新規 `require` ファイルがコピー対象に含まれているか確認
- [ ] パス（CSS・画像・リンク）が変わっていないか確認
- [ ] コピー順を確認

### 反映前の確認テンプレート

```
今回の反映内容を確認してください。

・新規作成：
・更新：
・削除：

不足しているファイル、コピー漏れ、
作業順序の問題、
新規ディレクトリの必要性、
require漏れ、
パス変更、
その他デプロイ時に問題になりそうな点があれば
反映前に指摘してください。
```

## デプロイ順序

1. ディレクトリ作成（必要な場合のみ）
2. 共通ファイル（`inc/`、`css/` など）
3. 記事・ページファイル
4. その他の更新ファイル

## デプロイ後

- [ ] ハブページ表示確認
- [ ] 子ページ表示確認
- [ ] 404が出ていないか確認
- [ ] CSS崩れなし
- [ ] CTA・ナビ・関連コンテンツが表示されている
- [ ] エラーログ確認（SSHで入れる場合）

## 注意事項

- ローカルに `php` がないため `php -l` 構文チェックは不可
- キャッシュ対策：CSS更新時は `Ctrl + Shift + R` またはクエリストリング（`?v=x.x.x`）で確認
- STG確認済みのファイルのみ本番に反映する
