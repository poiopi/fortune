# AutoLink v2 回帰テスト

`inc/auto-link.php`（文脈リンク自動挿入、以下 AutoLink）を変更したときに、
既知の不具合パターンが再発していないかを確認するためのツールです。

`inc/auto-link.php` は本ドキュメントの版数に合わせて **AutoLink v2** として
仕様固定します。今後の変更は「バグ修正」ではなく「仕様変更」として扱い、
変更前に本テストで現状の仕様が守られているかを確認したうえで着手してください。

**auto-link.php を変更したら、STG・本番へ反映する前に必ずこのテストを実行し、
「FAIL=0」になることを確認してから反映してください。**

## 設計思想（AutoLinkの仕様）

- **完全一致方式**：文脈（意味）を理解せず、文字列が含まれているかどうかだけで判定する単純な仕組み。
- **同義語のみ登録**：表記ゆれ・略称など、明確に同じ意味を指す語のみ登録する。
- **意味が曖昧な語は禁止**：日常語としても使われうる語（「世界」「正義」「恋人」「吉日」等）は、
  そのサイト内で意味が一意に定まらない限り登録しない。リンク数を増やすことより、
  クリックした先が必ず期待通りのページであることを優先する。
- **1文字禁止**：単語境界判定を行わないため、1文字キーワードは無関係な熟語を巻き込む。
- **selfリンク禁止**：各ページは自分自身の見出し語に対してリンクを生成しない。
- **HTML構造を壊さない**：属性値の中や既存の`<a>`タグの中は書き換えない。
- **ページ単位slug**：`autoLink()`の第2引数（自ページ除外用slug）は、記事ページ単位で一意にする
  （カテゴリ共通の値を使い回さない）。
- **回帰テスト必須**：変更時は必ず本テストを実行し、全項目PASSしてから反映する。

これらはAIが変わっても、開発者が変わっても、判断がブレないようにするための
恒久ルールです。判断に迷うキーワードは登録せず、「要検討」として保留すること。

## 実行方法

ローカルにPHP実行環境が必要です（`C:\php\php.exe` にインストール済み）。

```
C:\php\php.exe C:\Users\sende\fortune\test.life-fun.net\tools\debug\regression-test.php
```

最後に以下のように出力されればOKです。

```
結果: PASS=20 / FAIL=0 / 総テスト数=20
全項目PASS。反映して問題ありません。
```

`PASS`の合計数が変わったときは、テストケースが増減したということなので、
出力の`[PASS]`/`[FAIL]`の一覧（テスト名つき）で何が増減したかを確認できる。

`FAIL`が1件でもあれば、反映前に原因を直すこと。

## このツールがチェックしている内容

### no_single_char_keywords（1文字キーワード禁止）
`auto-link.php` の仕組みは単語境界判定を行わない単純な部分一致方式のため、
「木」「火」「甲」のような1文字キーワードを登録すると、無関係な熟語の中の1文字にまで
誤ってリンクを貼ってしまう。1文字キーワードが登録されていないかを確認する。

### slug_match（currentSlugとマップ側slugの整合性）
各記事ページが `autoLink($html, '自ページのslug')` の第2引数に渡している値と、
`auto-link.php` のマップ側に登録されている「自ページ除外用slug」が、記事ファイル
すべてについて一致しているかを確認する。

一致していないと、そのページが自分自身の見出し語を本文中で使ったときに、
自分自身へのリンクが誤って生成されてしまう（実際に過去、数秘術33ページ・
六曜/陰陽五行/特別な吉日の各ハブページで発生した）。

`_type-tpl.php`（MBTI）や `_sign-tpl.php`（西洋占星術）のように、共通テンプレートへ
`require` で処理を委譲しているページは、テンプレート側が `$type['slug']` のような
動的な値を使っていることを個別に確認済みとして扱う。

### self_link_exclusion（代表ページでの実行検証）
代表的なページについて、実際に `autoLink()` 関数を実行し、本当に自己リンクが
生成されないかを検証する。slug_matchが静的なコード比較なのに対し、こちらは
実際にロジックを動かして確認する。

### digit_collision（数字キーワード衝突確認）
「運命数3」が「運命数33」の一部を誤って奪ってしまうような、数字が続くキーワード同士の
衝突が起きていないかを確認する。文字数の降順ソートだけでは防げないパターンのため、
`auto-link.php` 側に否定先読み（`(?!\d)`）による対策が入っている。この対策が
機能しているかを実際にHTMLを生成して検証する。

### banned_words（過去に誤爆した語・単一漢字の再登録防止）
過去に実際の誤爆が確認された単語（世界・正義・恋人・戦車・吉日）と、単一漢字（木・火・土・金・水・
甲〜癸など）が、再び登録されていないかを確認する。

「吉日」は「特別な吉日」の部分文字列であり、「特別な吉日」の自己リンク除外が発動した
瞬間に未リンクのまま残ったテキストを「吉日」が横取りしてしまう不具合が実際に発生した
（digit_collisionと同じ根本原因の、数字以外での初めての実例）。

### particle_variations（助詞バリエーション確認）
「運命数11」「運命数が11」「運命数は11」「運命数の11」のように、日本語の助詞を挟んだ
表現がすべて正しくリンクされることを確認する。

### html_structure_protection（属性値の中は書き換えない）
`<img alt="愚者の絵">` のように、キーワードと同じ文字列がHTML属性値の中に偶然含まれる場合、
属性値の中に`<a>`タグを挿入してHTMLを破壊してしまわないかを確認する。

過去はこの保護が `<a>` タグにしか及んでおらず、属性値は無防備だった（実際に発見・修正済み）。
現在はタグ単位で分割し、テキスト部分だけを対象にすることで防いでいる。

### existing_link_protection（既存の<a>タグの中は再リンクしない）
本文中に既にある `<a href="...">キーワード</a>` のようなリンクの中身を、
さらに別のキーワードでリンク化してネストさせてしまわないかを確認する。

### 参考情報（PASS/FAIL判定はしない）
登録済みのキーワードが、対象カテゴリ以外のページの本文（`<p>`タグ内）で
使われていないかを機械的に検出する。ヒットしたものは「意味が一致しているかどうか」
まではこのツールでは判定できないため、一件ずつ目視で確認すること
（例：「戦車」が算命学「車騎星」の説明文で一般名詞として使われていたのはこの方法で発見した）。

## 注意事項

- このツールは開発用です。**本番環境（`C:\Users\sende\fortune\` 直下）には配置しないこと。**
- STGへの反映もVSCodeでの手動同期が必要。ローカルでこのテストがPASSしても、
  同期するまではSTG・本番のどちらにも反映されない。
- キーワードを追加・削除・変更、またはAutoLinkのロジックを変更したら、
  このテストを再実行してから反映する運用にする。
